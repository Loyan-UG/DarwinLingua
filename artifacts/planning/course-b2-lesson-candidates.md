# B2 Course Lesson Candidates

Date: 2026-05-31

## Purpose

This file preserves the reviewed B2 `CourseLesson` backlog before full multilingual course content generation.

Use it to keep B2 Course content comparable to common modern German coursebooks, Goethe/telc/Berufssprache B2 preparation, workplace communication, academic training contexts, and real German public-life communication while staying compatible with Darwin Lingua's German-first, linked-content architecture.

## Reference Anchors

The B2 plan aligns with:

- CEFR B2 expectations: clear and detailed expression on a wide range of topics, understanding complex texts on concrete and abstract topics, spontaneous interaction, explaining viewpoints, and presenting advantages and disadvantages.
- Goethe/telc-style B2 exam needs: reading, listening, writing, and speaking modules; response to input; opinion texts; paired discussion; presentation; argumentation; and direct interaction with another speaker.
- Berufssprache B2 needs: workplace meetings, customer communication, escalation, reporting, negotiation, feedback, project risks, and formal written documentation.
- Existing Darwin Lingua assets: B2 Grammar topics, B2 RoleplayScenario content, B2 Expressions, Talk Topics, Dialogues, Exercise sets, and future Exam Prep links.

## B2 Generation Rules

- Source `title`, `description`, `shortDescription`, `narrative`, `learningGoals`, `reviewSummary`, and `homeworkTask` must be German-first.
- Add learner-helper translations for `en`, `fa`, `ar`, `tr`, `ru`, `ckb`, `kmr`, `pl`, `ro`, and `sq`.
- B2 lessons should use longer, connected German examples and should train argument structure, nuance, register, and precision.
- Each lesson should normally include 4-6 concise German learning goals.
- B2 should train differentiated opinions, indirect criticism, negotiation, summaries, formal style, complex reading/listening, presentation, and workplace/public-service problem-solving.
- Course lessons must link to existing content where useful, but must not copy full Grammar/Expression/Dialogues/Roleplay/Exercise content.
- `linkedWordSlugs` may stay empty until a stable server-side word-slug convention exists.
- No sensitive/adult content belongs in this general B2 course path.
- Course imports for an existing path are cumulative: every package update must keep all reviewed lessons produced so far.
- Content production should start only after A1-C2 Course lesson planning files exist and are reviewed.

## Production Readiness Gate

Before generating full B2 lesson content from this plan:

- Confirm A1, A2, B1, B2, C1, and C2 Course planning files exist.
- Confirm the user explicitly starts content generation after planning is complete.
- Use Plan mode for the first content-generation prompt of each level.
- Generate only a small cumulative batch at a time.
- Keep all German source fields German-first and add learner-helper translations for all active learner languages.
- Require B2-level depth: longer examples, realistic constraints, differentiated reasoning, and clear register choices.
- Do not produce generic textbook filler; each lesson must include a concrete communicative outcome.
- Run JSON/language preflight and Course parser/import validation for each batch.
- Import to `darwinlingua_shared` only after validation.
- Verify PostgreSQL counts after import.
- Run lightweight Web/API smoke for at least one new lesson per batch.
- Take a database backup at the end of each completed CEFR level, not after every small batch.
