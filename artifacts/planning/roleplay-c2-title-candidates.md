# C2 RoleplayScenario Title Candidates

Date: 2026-05-26

## Purpose

This file preserves the reviewed C2 standalone `RoleplayScenario` title backlog before full multilingual content generation. Use it to avoid duplicate titles, ordinary lesson/dialogue tasks, English-first metadata, unsupported controlled values, and weak answer-choice design in future C2 roleplay batches.

## C2 Generation Rules

- Source `title`, `description`, and `learnerGoal` must be German-first.
- Add learner-language translations later for `en`, `fa`, `ar`, `tr`, `ru`, `ckb`, `kmr`, `pl`, `ro`, and `sq`.
- Keep C2 scenarios deterministic and scripted: no AI chat, no free-text scoring, and no learner-generated text sent to a model.
- C2 scenarios should usually have 12-16 turns with at least 5 answer-choice moments.
- C2 learner turns should require nuanced argumentation, register control, indirect criticism, rhetorical framing, concession, synthesis, and precise repair of misunderstandings.
- Recommended learner turns should appear in the scripted turn sequence.
- Answer choices must include at least one correct deterministic choice for validation, but learner-facing UI should emphasize plausible weaker/not-recommended alternatives instead of repeating the correct answer.
- Incorrect choices must be plausible but insufficient: too blunt, too evasive, too imprecise, insufficiently nuanced, missing concession, missing evidence, register-inappropriate, or strategically weak. Do not use obvious non-answer filler.
- Do not generate restricted sensitive content in the general C2 roleplay backlog. Socially risky topics may be discussed neutrally and without vulgar or explicit language.
- Use only importer-supported controlled values for `taskType`, `skillFocus`, `interactionMode`, and `register`.
- Use active topic keys, for example `everyday-life`, `social-and-relationships`, `shopping-and-services`, `appointments-and-health`, `public-services`, `transport-and-travel`, `education-and-training`, `customer-service`, `work-and-jobs`, `housing`, `business-communication`, `documents-and-administration`, and `meetings-and-presentations`.

- Run a preflight for required fields, active learner languages, controlled values, duplicate slugs, missing answer-choice moments, banned filler answers, English fallback markers, and German-first metadata.
- C2 scenarios should include nuanced weaker choices, not obviously irrelevant wrong answers.
- Run ContentOps Roleplay parser/application tests.
- Do not take a fresh `darwinlingua_shared` backup for every small content-only batch; take one at end-of-level checkpoints or before larger import/release operations.
- Import with `DarwinLingua.ImportTool --target shared --yes`.
- Verify PostgreSQL row counts and new active slugs.
- Smoke `/roleplays`, at least two detail pages, API detail, search, and admin report where applicable.
- Update this file and add a validation report under `artifacts/validation/`.
